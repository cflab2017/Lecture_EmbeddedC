## 문제 1 — 비트 명령 시퀀스
- 파일명: `homework_01/main.c`
- 핵심 개념: set(`|=`)·clear(`&= ~`)·toggle(`^=`)
- 플랫폼: 공통

### 요구사항
- 8비트 레지스터를 `0x00`에서 시작해 아래 명령을 순서대로 적용하고, **매 단계 후** 레지스터를 2진수 8자리로 출력한다.
  1. set bit 0
  2. set bit 2
  3. set bit 4
  4. toggle bit 2
  5. clear bit 0
  6. toggle bit 4
- 출력은 `step N: ` 다음에 8자리 2진수. 형식은 "예상 출력"과 글자 단위로 일치해야 한다.

### 예상 출력
```
step 1: 00000001
step 2: 00000101
step 3: 00010101
step 4: 00010001
step 5: 00010000
step 6: 00000000
```

### 힌트
- 마스크는 `(1u << n)`. clear는 반드시 `&= ~(1u << n)`(반전).
- 각 단계의 명령은 직전 결과 위에 누적된다(매번 0x00에서 다시 시작하지 않는다).
- 2진 출력은 최상위 비트(i=7)부터 i=0까지 `(v >> i) & 1`.

## 문제 2 — 세트된 비트 개수 세기
- 파일명: `homework_02/main.c`
- 핵심 개념: 비트 검사(`>> &`), 카운팅
- 플랫폼: 공통

### 요구사항
- 값 `0x00, 0x01, 0x0F, 0xAA, 0xFF` 각각에 대해 **1인 비트의 개수**를 세어 출력한다.
- 출력 형식은 `0xNN has K set bits`. 형식은 "예상 출력"과 글자 단위로 일치해야 한다.

### 예상 출력
```
0x00 has 0 set bits
0x01 has 1 set bits
0x0F has 4 set bits
0xAA has 4 set bits
0xFF has 8 set bits
```

### 힌트
- 8개 비트를 돌며 `(v >> i) & 1`을 더한다. 결과는 0 또는 1이라 그대로 더하면 개수가 된다.
- `v & (1<<i)`를 그대로 더하면 개수가 아니라 값이 더해지니 주의.
- 검산: `0xAA = 10101010` → 1이 4개, `0xFF` → 8개.
