## 문제 1 — 포인터로 가짜 레지스터 조작
- 파일명: `homework_01/main.c`
- 핵심 개념: 포인터 역참조(`*reg`), 비트 조작
- 플랫폼: 공통

### 요구사항
- 16비트 `fake_reg`를 `0x0000`에서 시작하고, 그 주소를 가리키는 `volatile uint16_t *reg`로만 조작한다.
- 다음을 순서대로 적용하고 **매 단계 후** `*reg`를 `0x%04X`로 출력한다.
  1. set bit 0
  2. set bit 12
  3. toggle bit 0
  4. set bit 4
- 출력 형식 `step N: 0xXXXX`는 "예상 출력"과 글자 단위로 일치해야 한다.

### 예상 출력
```
step 1: 0x0001
step 2: 0x1001
step 3: 0x1000
step 4: 0x1010
```

### 힌트
- 조작 대상은 `*reg`(값)이지 `reg`(주소)가 아니다.
- `BIT(12) = 1u << 12 = 0x1000`.
- `%04X`로 4자리 0 패딩.

## 문제 2 — 포인터로 두 값 교환(swap)
- 파일명: `homework_02/main.c`
- 핵심 개념: 주소 전달, 역참조
- 플랫폼: 공통

### 요구사항
- `uint8_t a = 0xAB; uint8_t b = 0xCD;`를 함수 호출 한 번으로 서로 바꾼다.
- 교환 전/후를 출력한다. 출력 형식은 "예상 출력"과 글자 단위로 일치해야 한다.

### 예상 출력
```
before: a = 0xAB, b = 0xCD
after : a = 0xCD, b = 0xAB
```

### 힌트
- `swap(a, b)`처럼 값으로 넘기면 원본이 안 바뀐다. `swap(&a, &b)`로 주소를 넘긴다.
- 함수 안에서 임시 변수 `tmp`를 두고 `*pa`, `*pb`를 교환한다.
