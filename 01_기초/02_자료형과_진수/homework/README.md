## 문제 1 — 8비트 값 진수표
- 파일명: `homework_01/main.c`
- 핵심 개념: 진수 표기(10/16/2진), 비트 추출(`>>`, `&`)
- 플랫폼: 공통

### 요구사항
- 값 `0, 1, 15, 16, 170, 255`를 차례로, 각 값을 한 줄에 **10진·16진·2진**으로 함께 출력한다.
- 10진은 폭 3에 오른쪽 정렬(`%3u`), 16진은 `0x` + 대문자 2자리(`0x%02X`), 2진은 8자리로 출력한다.
- 출력 형식은 아래 "예상 출력"과 글자 단위로 일치해야 한다.

### 예상 출력
```
dec=  0  hex=0x00  bin=00000000
dec=  1  hex=0x01  bin=00000001
dec= 15  hex=0x0F  bin=00001111
dec= 16  hex=0x10  bin=00010000
dec=170  hex=0xAA  bin=10101010
dec=255  hex=0xFF  bin=11111111
```

### 힌트
- 값을 배열에 담고 반복하면 코드가 짧아진다: `uint8_t values[] = {0,1,15,16,170,255};`
- 16진 2자리 0 패딩은 `%02X`. 빠뜨리면 `0x0F`가 `0xF`로 나온다.
- 2진은 표준 포맷이 없으니 직접 찍는다. 최상위 비트(i=7)부터 i=0까지 `(v >> i) & 1`로 한 비트씩.
- 검산: `170 = 0xAA = 10101010`, `15 = 0x0F = 00001111`.

## 문제 2 — uint8_t 카운터 랩어라운드
- 파일명: `homework_02/main.c`
- 핵심 개념: 부호 없는 자료형의 랩어라운드(255 → 0)
- 플랫폼: 공통

### 요구사항
- `uint8_t` 카운터를 **250에서 시작**해 1씩 **8번** 증가시키며, 매번 증가 후 값을 출력한다.
- 첫 줄에는 시작값을 `start:`로, 이후는 `step N:`(N은 1~8)으로 출력한다.
- 255 다음에 0으로 되돌아오는(wrap-around) 지점이 보여야 한다.

### 예상 출력
```
start: count = 250
step 1: count = 251
step 2: count = 252
step 3: count = 253
step 4: count = 254
step 5: count = 255
step 6: count = 0
step 7: count = 1
step 8: count = 2
```

### 힌트
- 카운터는 반드시 `uint8_t`로 선언한다. `int`로 하면 256, 257…로 커져 랩이 안 보인다.
- 8비트 unsigned는 0~255만 표현하므로 255 다음은 256이 아니라 0이다.
- 시작값 출력은 루프 전에 한 번, 그 뒤 루프에서 `count++` → 출력 순서.
