# 08. GPIO 입력 — 버튼/스위치 읽기

7편에서 핀으로 신호를 **내보냈다면(출력)**, 이번엔 핀으로 신호를 **읽어들입니다(입력)**. 버튼, 스위치, 센서의 디지털 신호를 받아 프로그램이 반응하게 만드는 일이죠. 8051에는 한 가지 특유의 함정이 있습니다. 포트가 **준쌍방향(quasi-bidirectional)** 이라, 입력으로 읽기 전에 그 핀에 **먼저 1을 써 두어야** 외부 신호를 제대로 읽을 수 있습니다.

이번 편은 P3.2에 연결된 버튼을 읽어 P1.0의 LED를 제어합니다. 버튼은 보통 **풀업 + 액티브 로우**(누르면 0)로 답니다. 더불어, 단순히 "눌린 동안 켜기"를 넘어 "**누르는 순간마다 토글**"하는 에지 검출 로직을 PC에서 검증합니다. 채터링(접점 떨림) 문제는 21편에서 본격적으로 다룹니다.

## 학습 목표
- 8051 준쌍방향 포트에서 입력을 읽기 전 1을 써야 하는 이유를 설명한다.
- `sbit`으로 입력 핀을 읽어 조건 분기한다.
- 풀업/액티브 로우 버튼의 신호(평소 1, 누르면 0)를 이해한다.
- 버튼 상태를 LED로 미러링한다.
- 에지 검출(1→0)로 "누르는 순간"만 감지하는 로직을 구현한다.

## 대상 환경
- 컴파일러: **Keil C51**.
- 디바이스: **AT89C52** (μVision 시뮬레이터).
- 검증: μVision 시뮬레이터 (Port 1/Port 3 창에서 핀 조작·관찰). 에지 로직은 PC `gcc`(`pc_test.c`).

## 핵심 개념

### 1) 준쌍방향 포트 — 읽기 전에 1을 써라
8051의 포트 핀은 출력 래치와 입력 버퍼를 공유합니다. 래치에 0을 써 두면 핀이 강하게 Low로 고정되어, 외부에서 High를 줘도 읽히지 않습니다. 그래서 **입력으로 쓸 핀에는 먼저 1을 써** 핀을 "풀어 둬야" 외부 신호가 반영됩니다.

```c
BTN = 1;            /* 입력 준비: 핀을 1(약한 풀업)로 만들어 둔다 */
if (BTN == 0) { }   /* 이제 외부에서 0 을 주면 읽힌다 */
```

> 리셋 직후 포트는 0xFF(모두 1)라 사실 기본은 입력 가능 상태지만, 같은 핀을 출력으로 쓰다가 입력으로 바꾼다면 반드시 1을 다시 써 줍니다. 습관으로 굳히는 게 안전합니다.

### 2) 핀 읽기
`sbit`으로 입력 핀에 이름을 주고 값을 비교합니다.

```c
sbit BTN = P3 ^ 2;
if (BTN == 0) { /* 눌림 */ }
```

포트 전체를 읽어 마스킹할 수도 있습니다: `if ((P3 & 0x04) == 0)` (P3.2 검사).

### 3) 풀업 + 액티브 로우 버튼
가장 흔한 버튼 회로는 한쪽을 GND에, 다른 쪽을 핀에 두고 **풀업 저항**으로 평소 High를 유지합니다.

| 상태 | 핀 값 |
|------|-------|
| 안 누름 | 1 (풀업) |
| 누름 | 0 (GND 연결) |

그래서 "눌림"의 판단은 `BTN == 0`입니다.

### 4) 버튼 미러링 vs 에지 검출
- **미러링(레벨)**: 눌린 **동안** LED ON. 단순히 현재 상태를 따른다.
- **에지 검출**: 누르는 **순간(1→0)** 에만 한 번 반응(예: 토글). 직전 상태와 현재 상태를 비교한다.

```c
if (prev_btn == 1 && btn == 0) {   /* 하강 에지 = 막 눌린 순간 */
    led = !led;                    /* 토글 */
}
prev_btn = btn;
```

에지 검출은 "한 번 누를 때 한 번만 동작"이 필요할 때 필수입니다(미러링은 누르고 있으면 계속 반응).

### 5) 채터링(바운스) — 맛보기
실제 버튼은 누르는 순간 접점이 수 ms간 떨려(chatter) 0↔1을 여러 번 오갑니다. 그러면 에지가 여러 번 감지되어 토글이 여러 번 일어날 수 있습니다. 해결책(디바운싱)은 짧은 지연 후 재확인하거나 일정 시간 안정될 때만 인정하는 것으로, **21편**에서 상태 머신과 함께 자세히 다룹니다.

## μVision 프로젝트 만들기
1. `Project → New µVision Project…` → 이름 `c51_gpio_in`.
2. Device: **Atmel → AT89C52**. STARTUP.A51 포함(Yes).
3. `main.c` 추가 후 아래 코드 복붙.
4. `Options for Target → Debug → Use Simulator`.
5. Rebuild(F7) → Start Debug(Ctrl+F5).
6. `Peripherals → I/O-Ports → Port 3`(입력 조작용)과 `Port 1`(LED 관찰용)을 연다. 시뮬레이터에서는 Port 3 창의 P3.2 체크박스를 직접 토글해 "버튼 누름"을 흉내 낸다.

## 예제로 보기

### 예제 1 — `main.c` : 버튼으로 LED 제어
버튼이 눌린 동안 LED를 켭니다(미러링).

```c
#include <reg52.h>

/*
 * 08강 예제 1 — 버튼 입력을 읽어 LED 제어.
 * 대상: AT89C52, 컴파일러: Keil C51.
 *
 * 버튼은 P3.2 에 액티브 로우(누르면 0, 풀업으로 평소 1)로 연결.
 * LED 는 P1.0 에 액티브 로우(0 이면 ON).
 *
 * 8051 의 포트는 '준쌍방향'이라, 입력으로 읽기 전에 그 핀에 1 을 먼저
 * 써 두어야 외부 신호를 제대로 읽을 수 있다.
 */

sbit BTN = P3 ^ 2;     /* 입력: 버튼 */
sbit LED = P1 ^ 0;     /* 출력: LED */

void main(void)
{
    BTN = 1;           /* 입력 준비: 읽기 전에 1 을 써 둔다 */

    while (1) {
        if (BTN == 0) {    /* 버튼 눌림(0) */
            LED = 0;       /* LED ON */
        } else {           /* 버튼 뗌(1) */
            LED = 1;       /* LED OFF */
        }
    }
}
```

### 예제 2 — `pc_test.c` : 누르는 순간 토글(에지 검출)
버튼 상태 시퀀스를 주고, 누르는 순간(1→0)마다 LED를 토글하는 로직을 검증합니다.

```c
#include <stdio.h>

/*
 * 08강 예제 2 — 누르는 '순간'마다 LED 를 토글하는 에지 검출 로직(PC 검증).
 * 버튼 상태 1=뗌, 0=눌림. 직전 상태와 비교해 1→0(하강 에지)일 때만 토글.
 * 실제 칩에서는 BTN 을 주기적으로 읽어 이 로직을 적용한다.
 */

static unsigned char update_led(unsigned char led,
                                unsigned char prev_btn,
                                unsigned char btn)
{
    if (prev_btn == 1 && btn == 0) {   /* 하강 에지 = 눌린 순간 */
        led = (unsigned char)!led;
    }
    return led;
}

int main(void)
{
    unsigned char seq[] = { 1, 1, 0, 0, 1, 0, 1, 0, 0, 1 };  /* 시간순 입력 */
    int n = (int)(sizeof(seq) / sizeof(seq[0]));
    unsigned char led = 0;
    unsigned char prev = 1;
    int t;

    for (t = 0; t < n; t++) {
        led = update_led(led, prev, seq[t]);
        printf("t=%d btn=%u led=%u\n", t, (unsigned)seq[t], (unsigned)led);
        prev = seq[t];
    }

    return 0;
}
```

- **예상 동작**: P3.2를 0으로 만들면 `Port 1`의 P1.0이 0(LED ON), 다시 1로 두면 P1.0이 1(LED OFF).

## 자주 하는 실수

### Q. 버튼을 눌러도 입력이 항상 0(또는 항상 같은 값)으로 읽혀요.
A. 8051 준쌍방향 포트의 함정입니다. 그 핀을 출력으로 0을 써 둔 적이 있으면 입력이 막힙니다. 읽기 전에 `BTN = 1;`로 핀을 풀어 두세요.

### Q. 한 번 눌렀는데 LED가 여러 번 토글돼요.
A. ① 미러링이 아니라 에지 검출을 쓰는데 채터링이 있으면 1→0이 여러 번 감지됩니다 → 디바운싱(21편). ② 루프가 너무 빨라 "누르고 있는 동안" 여러 번 도는 것을 토글로 착각한 경우 → 에지 검출로 바꾸세요.

### Q. `if (P3 & 0x04 == 0)`이 이상해요.
A. 우선순위 함정입니다. `==`가 `&`보다 먼저라 `0x04 == 0`(거짓=0)이 먼저 계산됩니다. 괄호로: `if ((P3 & 0x04) == 0)`. 또는 `sbit`으로 `if (BTN == 0)`이 더 명확합니다.

### Q. 풀업이 없으면요?
A. 입력 핀이 어디에도 연결 안 되면 값이 떠다녀(floating) 무작위로 읽힙니다. 외부 풀업/풀다운 저항이나 (가능한 MCU면) 내부 풀업을 써서 기본 레벨을 정해야 합니다. 8051 포트는 내부 약한 풀업이 있어 평소 1로 유지됩니다.

## 정리
- 8051 준쌍방향 포트는 입력으로 읽기 전 핀에 1을 써 둬야 한다.
- 풀업+액티브 로우 버튼은 평소 1, 누르면 0. 판단은 `BTN == 0`.
- 미러링은 현재 상태를 따르고, 에지 검출은 "누르는 순간"만 잡는다.
- 에지 검출은 직전 상태와 현재 상태를 비교한다(`prev==1 && cur==0`).
- 접점 떨림(채터링)은 디바운싱으로 해결한다(21편).
