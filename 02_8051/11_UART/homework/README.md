# 과제 - 11. UART 시리얼 통신

문제 1은 **C51(AT89C52, μVision 시뮬레이터)**, 문제 2는 **순수 로직(PC `gcc` 검증)** 입니다.

---

## 문제 1 — UART 에코(소문자는 대문자로)
- 파일명: `homework_01/main.c`
- 핵심 개념: UART 수신(RI/SBUF), 송신(TI/SBUF), 문자 처리
- 플랫폼: C51 (AT89C52)

### 요구사항
- UART를 9600bps·8N1로 초기화한다(SCON, Timer1 Mode2, TH1=0xFD, 11.0592MHz 가정).
- 한 글자를 수신하면, 소문자(`a`~`z`)는 대문자로 바꿔 그대로 되돌려 보낸다(에코).
- 그 외 문자는 변환 없이 그대로 되돌려 보낸다.

### 예상 동작 (시뮬레이터)
- `UART #1` 창에 `abc`를 입력하면 `ABC`가, `Hi 8051`을 입력하면 `HI 8051`이 되돌아온다.

### 힌트
- 수신: `while (RI == 0); RI = 0; c = SBUF;`. 송신: `SBUF = c; while (TI == 0); TI = 0;`.
- 대문자 변환: `if (c >= 'a' && c <= 'z') c = c - 'a' + 'A';`.
- `REN`(수신 허용)을 꼭 켤 것 — `SCON = 0x50`에 포함되어 있다.

## 문제 2 — UART 8N1 프레임 펼치기 (PC)
- 파일명: `homework_02/main.c`
- 핵심 개념: 시리얼 프레임 구조(start/data/stop), LSB 우선 전송
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 문자열 `"Hi"`의 각 글자에 대해, 라인에 실리는 8N1 비트열을 출력한다.
- 순서는 **start(0) → 데이터 8비트(LSB부터) → stop(1)**. 데이터는 LSB(b0)부터다.
- 출력 형식은 "예상 출력"과 글자 단위로 일치해야 한다(start와 stop은 양옆에 공백으로 구분).

### 예상 출력
```
'H' 0x48: 0 00010010 1
'i' 0x69: 0 10010110 1
```

### 힌트
- 데이터 비트는 `for (b = 0; b < 8; b++) printf("%d", (c >> b) & 1);`로 LSB부터 출력.
- 줄 형식: `printf("'%c' 0x%02X: 0 ", c, c);` → 데이터 8비트 → `printf(" 1\n");`.
- `'H'=0x48`의 LSB 우선 데이터는 `00010010`임을 손으로 확인해 보자.
