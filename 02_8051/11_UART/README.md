# 11. UART 시리얼 통신

지금까지 LED와 버튼으로 칩과 대화했지만, 값을 "사람이 읽는 글자"로 주고받지는 못했습니다. 디버깅 메시지를 찍거나 센서 값을 PC로 보내려면 통신 수단이 필요합니다. 가장 기본이 **UART(시리얼)** 입니다. 선 두 가닥(TXD/RXD)으로 칩과 PC가 텍스트를 주고받습니다.

UART는 **비동기(asynchronous)** 통신입니다. 클럭선이 따로 없어서, 송수신 양쪽이 미리 **보율(baud rate)** 을 똑같이 약속해 두고 그 속도로 비트를 싣고 읽습니다. 8051은 **SCON** 으로 시리얼 모드를, **Timer1** 으로 보율 클럭을 만들고, **SBUF** 라는 한 바이트짜리 창구로 데이터를 넣고 뺍니다. 이번 편은 9600bps·8N1로 PC에 문자열을 보내고, 받은 문자를 처리하는 법을 익힙니다.

## 학습 목표
- 동기/비동기 통신의 차이와 UART 프레임(start·data·stop)을 설명한다.
- `SCON`으로 시리얼 모드(Mode1, 8비트 UART)와 수신 허용(REN)을 설정한다.
- `Timer1`(Mode2 자동 리로드)으로 보율을 만들고 `TH1` 리로드 값을 계산한다.
- `SBUF`/`TI`/`RI`로 한 바이트를 송신·수신한다.
- UART용 크리스털로 11.0592MHz를 쓰는 이유를 안다.

## 대상 환경
- 컴파일러: **Keil C51**.
- 디바이스: **AT89C52** (μVision 시뮬레이터). UART는 **11.0592MHz** 크리스털 가정.
- 검증: μVision 시뮬레이터 (`View → Serial Windows → UART #1`에서 송수신 관찰). 보율 계산은 PC `gcc`(`pc_test.c`).

> 9편 타이머는 시간 계산 편의를 위해 12MHz를 가정했지만, UART는 보율 오차를 없애기 위해 **11.0592MHz**를 쓰는 게 정석입니다(아래 3) 참고).

## 핵심 개념

### 1) UART란 — 비동기 시리얼
데이터를 한 비트씩 한 선으로 보내는 방식이 **시리얼**입니다. UART는 별도 클럭선 없이 동작하는 **비동기** 방식이라, 양쪽이 같은 **보율**(초당 비트 수, 예: 9600bps)로 맞춰 두는 것이 전제입니다. 연결은 보통 세 가닥: **TXD**(송신, 8051은 P3.1), **RXD**(수신, P3.0), **GND**.

| | 동기(SPI/I2C) | 비동기(UART) |
|---|---------------|---------------|
| 클럭선 | 있음 | 없음 |
| 속도 합의 | 클럭으로 자동 | 보율을 미리 약속 |
| 선 개수 | 많음 | 적음(TXD/RXD/GND) |

### 2) 프레임 — start / data / stop (8N1)
유휴 상태의 라인은 High(1)입니다. 한 바이트를 보낼 때 라인에 실리는 순서는:

```
유휴(1) → [start=0] → [d0 d1 d2 d3 d4 d5 d6 d7] → [stop=1] → 유휴(1)
                       ↑ 데이터는 LSB(d0)부터!
```

- **start 비트**: 항상 0. High였던 라인을 Low로 떨어뜨려 "시작"을 알린다(수신측 동기 잡기).
- **데이터 비트**: 8비트를 **LSB부터** 보낸다.
- **패리티**: 없음(N).
- **stop 비트**: 항상 1.

데이터 8비트, 패리티 없음(None), 정지 1비트 → 흔히 **"8N1"** 이라 부릅니다. 과제 2에서 한 바이트가 어떤 비트열로 나가는지 직접 펼쳐 봅니다.

### 3) 보율과 Timer1 — 왜 11.0592MHz인가
8051은 **Timer1의 오버플로 속도**로 보율 클럭을 만듭니다. Timer1을 **Mode2(8비트 자동 리로드)** 로 두고 `TH1`에 리로드 값을 넣습니다. SMOD=0일 때:

```
보율  = Fosc / (384 × (256 - TH1))
→ n = 256 - TH1 = Fosc / (384 × 보율)
```

11.0592MHz에서 9600bps:

```
n   = 11,059,200 / (384 × 9600) = 3   (정수로 딱 떨어진다!)
TH1 = 256 - 3 = 253 = 0xFD
```

11.0592MHz는 표준 보율(1200/2400/4800/9600…)에서 `n`이 **정수**로 떨어져 오차가 0입니다. 반면 12MHz는 9600에서 `n = 3.26…` → 정수 3으로 반올림하면 실제 약 10417bps, **오차 약 8.5%** 라 통신이 깨집니다. 그래서 UART용 보드는 11.0592MHz 크리스털을 즐겨 씁니다.

### 4) SCON — 시리얼 제어 레지스터
| 비트 | 이름 | 역할 |
|------|------|------|
| SM0/SM1 | 모드 선택 | `01`=Mode1(8비트 UART, 가변 보율) |
| SM2 | 멀티프로세서 | 보통 0 |
| REN | 수신 허용 | 1이면 수신 가능 |
| TB8/RB8 | 9번째 비트 | Mode2/3에서 사용 |
| TI | 송신 완료 플래그 | 한 바이트 다 보내면 1(소프트웨어가 클리어) |
| RI | 수신 완료 플래그 | 한 바이트 다 받으면 1(소프트웨어가 클리어) |

표준 설정 **`SCON = 0x50`** → `0101 0000`: SM0=0, SM1=1(Mode1), REN=1(수신 허용), 나머지 0.

### 5) SBUF와 송수신 흐름
`SBUF`는 같은 이름이지만 **쓰면 송신용, 읽으면 수신용**인 한 바이트 레지스터입니다.

**송신**: `SBUF`에 쓰면 하드웨어가 비트를 밀어내고, 다 보내면 `TI=1`. 소프트웨어가 `TI`를 기다렸다 0으로 지운다.

```c
void uart_tx(unsigned char c) {
    SBUF = c;            /* 송신 시작 */
    while (TI == 0);     /* 완료까지 대기 */
    TI = 0;              /* 플래그 클리어 */
}
```

**수신**: 한 바이트가 들어오면 `RI=1`. 소프트웨어가 `SBUF`를 읽고 `RI`를 지운다.

```c
unsigned char uart_rx(void) {
    while (RI == 0);     /* 수신까지 대기 */
    RI = 0;              /* 플래그 클리어 */
    return SBUF;
}
```

> `TI`/`RI`는 하드웨어가 **세우기만** 하고 내려 주지 않습니다. 소프트웨어가 매번 0으로 클리어해야 다음 바이트를 처리할 수 있습니다.

### 6) (참고) `printf`를 UART로 보내기
Keil C51의 `printf`는 내부적으로 `putchar` 한 글자를 출력합니다. `putchar`를 UART 송신으로 바꿔 두면 `printf`가 그대로 시리얼로 나갑니다.

```c
char putchar(char c) {     /* Keil C51 라이브러리가 호출 */
    SBUF = c;
    while (TI == 0);
    TI = 0;
    return c;
}
```

> 8051은 RAM이 작아 `printf`가 무겁습니다. 디버깅 외에는 직접 만든 `uart_tx`/`uart_puts`가 가볍습니다.

## μVision 프로젝트 만들기
1. `Project → New µVision Project…` → 이름 `c51_uart`.
2. Device: **Atmel → AT89C52**. STARTUP.A51 포함(Yes).
3. `Options for Target → Target` 탭의 Xtal(MHz)을 **11.0592**로 설정(보율 정확도).
4. `main.c` 추가 후 아래 코드 복붙.
5. `Options for Target → Debug → Use Simulator`.
6. Rebuild(F7) → Start Debug(Ctrl+F5).
7. `View → Serial Windows → UART #1` 창을 연다(송수신 텍스트가 여기 표시됨).

## 예제로 보기

### 예제 1 — `main.c` : UART로 문자열 보내기
9600bps·8N1로 초기화하고, "Hello, 8051!"을 반복 송신합니다.

```c
#include <reg52.h>

/*
 * 11강 예제 1 — UART 로 PC 에 문자열 보내기(9600bps, 8N1).
 * 대상: AT89C52, 컴파일러: Keil C51. UART 용 11.0592MHz 크리스털 가정.
 *
 * 비동기 시리얼(UART)은 클럭선 없이, 양쪽이 약속한 '보율(baud)'로 비트를
 * 주고받는다. 8051 은 Timer1 의 오버플로로 보율 클럭을 만든다.
 *   9600bps @ 11.0592MHz, SMOD=0 → TH1 = 256 - 11059200/(384*9600) = 0xFD
 *
 * 송신 흐름: SBUF 에 한 바이트를 쓰면 하드웨어가 비트를 밀어내고, 다 보내면
 * TI(송신 완료 플래그)를 1 로 세운다. 소프트웨어가 TI 를 기다렸다 0 으로 지운다.
 */

/* UART 를 9600bps, 8비트, 정지 1, 패리티 없음(8N1)으로 초기화. */
void uart_init(void)
{
    SCON = 0x50;          /* SM1=1(Mode1, 8비트 UART) + REN=1(수신 허용) */
    TMOD &= 0x0F;         /* Timer1 비트만 클리어(Timer0 설정 보존) */
    TMOD |= 0x20;         /* Timer1 = Mode2(8비트 자동 리로드) → 보율 생성 */
    TH1  = 0xFD;          /* 리로드 값: 9600bps @ 11.0592MHz */
    TL1  = 0xFD;
    TR1  = 1;             /* Timer1 시작 */
}

/* 한 바이트 송신(완료까지 대기). */
void uart_tx(unsigned char c)
{
    SBUF = c;             /* 보낼 바이트를 버퍼에 쓰면 송신 시작 */
    while (TI == 0) {     /* 송신 완료(TI=1)까지 대기 */
    }
    TI = 0;               /* 다음 송신을 위해 플래그 클리어 */
}

/* 널 종료 문자열 송신. */
void uart_puts(const char *s)
{
    while (*s) {
        uart_tx((unsigned char)*s++);
    }
}

/* 관찰하기 좋게 송신 사이에 잠깐 쉰다(정밀할 필요 없는 소프트웨어 지연). */
static void delay(void)
{
    unsigned int i;
    for (i = 0; i < 60000; i++) {
    }
}

void main(void)
{
    uart_init();
    while (1) {
        uart_puts("Hello, 8051!\r\n");   /* Serial 창에서 한 줄씩 보인다 */
        delay();
    }
}
```

### 예제 2 — `pc_test.c` : 보율 리로드(TH1) 계산 검증
표준 보율별로 Timer1 리로드 값을 계산해, 11.0592MHz에서 정수로 떨어지는지 확인합니다.

```c
#include <stdio.h>

/*
 * 11강 예제 2 — UART 보율(baud)에 맞는 Timer1 리로드(TH1) 계산 검증.
 * 11.0592MHz 크리스털, SMOD=0, Timer1 Mode2(8비트 자동 리로드) 기준.
 *   n   = Fosc / (384 * baud)   (Timer1 이 한 번 오버플로될 때까지 세는 수)
 *   TH1 = 256 - n
 * 11.0592MHz 는 표준 보율에서 n 이 '정수'로 떨어져 보율 오차가 0 이다. 그래서
 * UART 용 크리스털로 12MHz 대신 11.0592MHz 를 즐겨 쓴다(12MHz 는 9600 에서 약 8.5%
 * 오차가 나 통신이 깨진다).
 */
int main(void)
{
    unsigned long fosc = 11059200UL;       /* 11.0592 MHz */
    unsigned int bauds[] = { 1200, 2400, 4800, 9600 };
    int count = (int)(sizeof(bauds) / sizeof(bauds[0]));
    int k;

    for (k = 0; k < count; k++) {
        unsigned int baud = bauds[k];
        unsigned int n = (unsigned int)(fosc / (384UL * baud));
        unsigned char th1 = (unsigned char)(256 - n);
        printf("%5u baud -> n=%3u  TH1=0x%02X\n", baud, n, th1);
    }

    return 0;
}
```

## 시뮬레이터로 확인하기
- `View → Serial Windows → UART #1` 창을 연 채로 F5(실행).
- **예상 동작**: UART #1 창에 `Hello, 8051!`이 한 줄씩 반복 출력된다.
- 송신 직후 `SCON`의 `TI` 비트가 1로 섰다가 코드가 0으로 지우는 모습을 레지스터 창에서 볼 수 있다.
- (수신 실험) `Peripherals` 없이도 UART #1 창에 키를 입력하면 RXD로 들어간다. 에코는 과제 1에서 다룬다.

## 자주 하는 실수

### Q. 한 글자는 나오는데 두 번째부터 멈춰요.
A. `TI`(송신 완료 플래그)를 클리어하지 않았을 가능성이 큽니다. 하드웨어는 `TI`를 1로 세우기만 합니다. 매 송신 후 `TI = 0;`으로 직접 지워야 다음 `while (TI == 0)`이 동작합니다. 수신은 `RI`도 마찬가지.

### Q. 글자가 깨져서(□나 이상한 문자) 나와요.
A. 보율이 안 맞습니다. ① 시뮬레이터/터미널의 보율을 9600으로 맞췄는지, ② Xtal을 11.0592MHz로 설정했는지 확인하세요. 12MHz로 두면 `TH1=0xFD`라도 실제 보율이 ~10417bps라 깨집니다.

### Q. 송신은 되는데 수신이 안 돼요.
A. `SCON`의 **REN**(수신 허용) 비트를 안 켰을 수 있습니다. `SCON = 0x50`은 REN=1을 포함합니다. 또 `RI`를 읽은 뒤 클리어했는지 확인하세요.

### Q. `TH1`만 넣었는데 보율이 이상해요.
A. Timer1을 **Mode2(자동 리로드)** 로 설정했는지(`TMOD |= 0x20`), `TR1 = 1`로 **시작**했는지 확인하세요. Mode2가 아니면 매번 리로드가 안 돼 보율이 틀어집니다.

### Q. `printf`가 너무 무겁거나 안 나와요.
A. Keil C51에서 `printf`를 UART로 보내려면 `putchar`를 재정의해야 합니다(핵심 개념 6). 그래도 8051에선 무거우니, 디버깅 외에는 직접 만든 `uart_tx`/`uart_puts`를 쓰세요.

## 정리
- UART는 클럭선 없는 비동기 시리얼. 양쪽이 같은 **보율**로 약속해야 한다.
- 프레임은 start(0) → 데이터 8비트(**LSB부터**) → stop(1), 패리티 없음이면 **8N1**.
- `SCON = 0x50`으로 Mode1·REN, `Timer1` Mode2(`TMOD |= 0x20`)로 보율 생성.
- 보율 리로드 `TH1 = 256 - Fosc/(384×보율)`. 11.0592MHz면 표준 보율이 정수로 떨어진다.
- `SBUF`에 쓰면 송신·읽으면 수신. `TI`/`RI`는 소프트웨어가 매번 클리어한다.
