# 과제 - 07. GPIO 출력 — LED 점멸

문제 1은 **C51(AT89C52, μVision 시뮬레이터)**, 문제 2는 **순수 로직(PC `gcc` 검증)** 입니다.

---

## 문제 1 — 두 LED 교대 점멸
- 파일명: `homework_01/main.c`
- 핵심 개념: 포트 비트 쓰기, `while(1)`, 지연
- 플랫폼: C51 (AT89C52)

### 요구사항
- P1.0(LED0)과 P1.1(LED1)을 **항상 반대 상태**로 유지하며 번갈아 점멸한다(액티브 로우).
- 매 주기 지연 후 역할을 바꾼다(LED0 ON↔LED1 ON).

### 예상 동작 (시뮬레이터)
- Port 1에서 P1.0과 P1.1이 번갈아 0(ON)/1(OFF)이 된다.
- LED0이 켜지면 LED1은 꺼지고, 다음 주기엔 반대.

### 힌트
- `sbit LED0 = P1^0; sbit LED1 = P1^1;`을 전역에 선언.
- 한쪽을 `state`로, 다른 쪽을 `!state`로 두면 항상 반대가 보장된다.
- `delay()`를 넣어 관찰 가능하게 한다.

## 문제 2 — 두 칸짜리 흐르는 불빛 (PC)
- 파일명: `homework_02/main.c`
- 핵심 개념: 다중 비트 마스크, 액티브 로우 반전
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 인접한 LED 2개를 동시에 켜며 `pos`를 0→6으로 옮긴 포트 값을 출력한다(액티브 로우).
- 포트 값 = `~((1<<pos) | (1<<(pos+1)))`.
- 출력 형식 `pos N: 0xNN`은 "예상 출력"과 글자 단위로 일치해야 한다.

### 예상 출력
```
pos 0: 0xFC
pos 1: 0xF9
pos 2: 0xF3
pos 3: 0xE7
pos 4: 0xCF
pos 5: 0x9F
pos 6: 0x3F
```

### 힌트
- 마스크는 두 비트를 OR로 묶는다. 액티브 로우라 포트엔 그 반전(`~`)을 쓴다.
- `pos`는 0~6까지만(7이면 `1<<8`로 포트를 벗어남).
