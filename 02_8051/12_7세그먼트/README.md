# 12. 7세그먼트 표시 (멀티플렉싱)

LED 하나로 깜빡임을 만들었다면, 이번엔 **사람이 읽을 수 있는 숫자**를 표시합니다. 7-세그먼트 디스플레이는 막대 모양 LED 7개(a~g)와 소수점(dp)으로 0~9, 일부 문자를 그립니다. 핵심은 두 가지입니다. ① 숫자를 "어떤 막대를 켤지"의 비트 패턴으로 바꾸는 **룩업 테이블**, ② 자리(digit)가 여러 개일 때 핀을 아끼면서 모두 켜 보이게 하는 **멀티플렉싱(동적 구동)** 입니다.

이번 편은 6~7편의 포트 출력 위에, 룩업 테이블과 시간 분할(time-division)이라는 새 도구를 얹습니다. 4자리 숫자 "1234"를 한 자리씩 아주 빠르게 번갈아 켜서, 잔상 효과로 네 자리가 동시에 켜진 것처럼 보이게 만듭니다.

> **시뮬레이터 메모**: μVision 시뮬레이터에는 7-세그먼트 표시 장치가 그래픽으로 없습니다. 그래서 이 편은 "세그먼트 패턴(P2)과 자리 선택(P0)이 올바른 값으로 순환하는지"를 Port 창/Logic Analyzer로 확인하고, **숫자 → 세그먼트 변환 자체는 `pc_test.c`로 PC에서 검증**합니다.

## 학습 목표
- 7-세그먼트의 세그먼트 배치(a~g, dp)와 공통 음극/양극 차이를 설명한다.
- 0~9를 세그먼트 비트 패턴으로 바꾸는 룩업 테이블을 만든다.
- 자리 선택과 세그먼트 출력을 분리해 여러 자리를 구동한다.
- 멀티플렉싱(시간 분할 + 잔상)으로 4자리를 동시에 켜 보이게 한다.
- 잔상(ghosting)을 막는 "끄고 → 바꾸고 → 켜기" 순서를 적용한다.

## 대상 환경
- 컴파일러: **Keil C51**.
- 디바이스: **AT89C52** (μVision 시뮬레이터).
- 검증: μVision 시뮬레이터 (Port 0 / Port 2 창, Logic Analyzer). 숫자→코드 변환은 PC `gcc`(`pc_test.c`).

## 핵심 개념

### 1) 7-세그먼트 구조와 배선
세그먼트 7개(a~g)와 소수점(dp)이 한 자리를 이룹니다. 모든 세그먼트의 한쪽 다리가 공통으로 묶이는데, 묶는 극성에 따라 두 종류입니다.

| 종류 | 공통 단자 | 세그먼트 ON 조건 | 비고 |
|------|-----------|------------------|------|
| 공통 음극 (common cathode) | 모든 LED의 (−)를 GND로 | 세그먼트 핀 = **1** | 이 편의 가정 |
| 공통 양극 (common anode) | 모든 LED의 (+)를 VCC로 | 세그먼트 핀 = **0** | 과제 1에서 다룸 |

7편의 액티브 하이/로우와 같은 이야기입니다. 이 편은 **공통 음극**(비트=1이 ON)을 가정합니다.

### 2) 세그먼트 비트 배치
세그먼트를 포트 비트에 다음처럼 매핑합니다(이 편 약속).

```
   aaa          비트:  a=0  b=1  c=2  d=3  e=4  f=5  g=6  dp=7
  f   b         숫자 0 = a,b,c,d,e,f ON, g OFF
  f   b                = 0b00111111 = 0x3F
   ggg
  e   c
  e   c
   ddd  (dp)
```

### 3) 0~9 룩업 테이블
숫자마다 켤 세그먼트가 정해져 있으니, 미리 표로 만들어 두고 `SEG_CODE[n]`으로 꺼내 씁니다. 매번 비트를 계산하는 것보다 빠르고 읽기 쉽습니다.

| 숫자 | 켜는 세그먼트 | 코드 | 숫자 | 켜는 세그먼트 | 코드 |
|------|---------------|------|------|---------------|------|
| 0 | a b c d e f | `0x3F` | 5 | a c d f g | `0x6D` |
| 1 | b c | `0x06` | 6 | a c d e f g | `0x7D` |
| 2 | a b d e g | `0x5B` | 7 | a b c | `0x07` |
| 3 | a b c d g | `0x4F` | 8 | a b c d e f g | `0x7F` |
| 4 | b c f g | `0x66` | 9 | a b c d f g | `0x6F` |

### 4) 자리 선택 — 핀을 아끼는 이유
4자리를 각각 7핀씩 따로 연결하면 28핀이 필요합니다. 대신 **세그먼트 핀(a~g)은 4자리가 공유**하고, "지금 어느 자리를 켤지"만 자리 선택 핀으로 고릅니다. 그러면 7(세그먼트) + 4(자리) = 11핀으로 충분합니다. 단, 세그먼트선을 공유하므로 **한 순간에는 한 자리만** 켤 수 있습니다.

### 5) 멀티플렉싱과 잔상
한 자리만 켤 수 있다면 어떻게 네 자리를 다 보여 줄까요? **빠르게 번갈아** 켭니다. 자리0 → 자리1 → 자리2 → 자리3 → 다시 자리0 … 을 수백 µs 간격으로 반복하면, 눈은 잔상 때문에 네 자리가 동시에 켜진 것으로 인식합니다.

순서가 중요합니다. 자리를 켠 채 세그먼트 패턴을 바꾸면, 바뀌는 순간 엉뚱한 패턴이 잠깐 그 자리에 비쳐 **잔상(ghosting)** 이 생깁니다. 그래서 항상:

```
1) 모든 자리 끄기  →  2) 새 세그먼트 패턴 쓰기  →  3) 해당 자리만 켜기
```

## μVision 프로젝트 만들기
1. `Project → New µVision Project…` → 이름 `c51_seg7`.
2. Device: **Atmel → AT89C52**.
3. STARTUP.A51 포함(Yes).
4. `main.c` 추가 후 아래 코드 복붙.
5. `Options for Target → Debug → Use Simulator`.
6. Rebuild(F7) → Start Debug(Ctrl+F5).
7. 관찰: `Peripherals → I/O-Ports → Port 0`(자리 선택)과 `Port 2`(세그먼트)를 띄운다. 또는 `View → Analysis Windows → Logic Analyzer`에 `P0.0~P0.3`, `P2.0~P2.6`을 추가한다.

## 예제로 보기

### 예제 1 — `main.c` : 4자리 멀티플렉싱
"1234"를 한 자리씩 빠르게 번갈아 켭니다. 잔상 방지를 위해 매 자리마다 끄고→쓰고→켭니다.

```c
#include <reg52.h>

/*
 * 12강 예제 — 4자리 7-세그먼트 동적 구동(멀티플렉싱).
 * 대상: AT89C52, 컴파일러: Keil C51.
 *
 * 배선 가정(공통 음극, common cathode):
 *   - 세그먼트 a~g, dp 는 P2 에 연결. 비트=1 이면 그 세그먼트 ON.
 *       a=P2.0  b=P2.1  c=P2.2  d=P2.3  e=P2.4  f=P2.5  g=P2.6  dp=P2.7
 *   - 자리 선택은 P0 하위 4비트. 비트=1 이면 그 자리를 켠다(자리당 1비트).
 *       digit0=P0.0  digit1=P0.1  digit2=P0.2  digit3=P0.3
 *
 * 멀티플렉싱: 한 순간에는 한 자리만 켜고, 4자리를 아주 빠르게 돌며 켠다.
 * 사람 눈의 잔상(persistence of vision) 덕분에 4자리가 동시에 켜진 것처럼 보인다.
 */

#define SEG_PORT  P2     /* 세그먼트 패턴 출력 */
#define DIG_PORT  P0     /* 자리 선택 */

/* 공통 음극용 0~9 세그먼트 코드 (비트=1 → 세그먼트 ON).
 * 비트 순서: a(0) b(1) c(2) d(3) e(4) f(5) g(6) dp(7) */
static const unsigned char SEG_CODE[10] = {
    0x3F, /* 0 */ 0x06, /* 1 */ 0x5B, /* 2 */ 0x4F, /* 3 */ 0x66, /* 4 */
    0x6D, /* 5 */ 0x7D, /* 6 */ 0x07, /* 7 */ 0x7F, /* 8 */ 0x6F  /* 9 */
};

/* 짧은 소프트웨어 지연(자리 유지 시간). 정확한 시간은 9편 타이머 참고. */
static void delay(unsigned int count)
{
    unsigned int i;
    for (i = 0; i < count; i++) {
        /* 시간 보내기 */
    }
}

void main(void)
{
    /* 표시할 4자리. digits[0] 이 가장 왼쪽(자리0). 여기서는 "1234". */
    unsigned char digits[4] = { 1, 2, 3, 4 };
    unsigned char d;

    while (1) {
        for (d = 0; d < 4; d++) {
            DIG_PORT = 0x00;                       /* 1) 모든 자리 끄기(잔상 방지) */
            SEG_PORT = SEG_CODE[digits[d]];        /* 2) 이 자리의 세그먼트 패턴 */
            DIG_PORT = (unsigned char)(1u << d);   /* 3) 이 자리만 켜기 */
            delay(150);                            /* 4) 잠깐 유지 후 다음 자리 */
        }
    }
}
```

### 예제 2 — `pc_test.c` : 룩업 테이블 검산
하드웨어 없이, 0~9가 어떤 세그먼트를 켜는지 글자로 그려 봅니다(대문자=ON, `-`=OFF).

```c
#include <stdio.h>

/*
 * 12강 PC 검산 — 0~9 의 공통 음극 세그먼트 코드와 켜지는 세그먼트를 출력한다.
 * 하드웨어 없이 룩업 테이블의 정확성만 확인한다.
 */

static const unsigned char SEG_CODE[10] = {
    0x3F, 0x06, 0x5B, 0x4F, 0x66, 0x6D, 0x7D, 0x07, 0x7F, 0x6F
};

/* 켜진 세그먼트는 a~g 글자로, 꺼진 세그먼트는 '-' 로 표시한다. */
static void print_seg(unsigned char code)
{
    static const char names[7] = { 'A', 'B', 'C', 'D', 'E', 'F', 'G' };
    int i;
    for (i = 0; i < 7; i++) {
        putchar((code & (1u << i)) ? names[i] : '-');
    }
}

int main(void)
{
    int n;
    for (n = 0; n <= 9; n++) {
        printf("%d -> 0x%02X  ", n, SEG_CODE[n]);
        print_seg(SEG_CODE[n]);
        putchar('\n');
    }
    return 0;
}
```

## 시뮬레이터로 확인하기
- Port 2(세그먼트)가 자리 순서대로 `0x06`(1) → `0x5B`(2) → `0x4F`(3) → `0x66`(4) … 으로 바뀐다.
- Port 0(자리 선택)이 `0x01` → `0x02` → `0x04` → `0x08` 으로 한 비트씩 이동하며, P2의 패턴과 박자가 맞는다.
- Logic Analyzer로 보면, 어느 한 순간 P0 핀 하나만 High이고 그때 P2에 해당 자리의 숫자 패턴이 실린다.
- **예상 동작**: (실제 디스플레이라면) 4자리에 "1234"가 안정적으로 표시. 시뮬에서는 P0/P2 값의 동기화된 순환으로 확인.

## PC에서 검증하기

```
gcc pc_test.c -o pc_test && ./pc_test
```

예상 출력(stdout):

```
0 -> 0x3F  ABCDEF-
1 -> 0x06  -BC----
2 -> 0x5B  AB-DE-G
3 -> 0x4F  ABCD--G
4 -> 0x66  -BC--FG
5 -> 0x6D  A-CD-FG
6 -> 0x7D  A-CDEFG
7 -> 0x07  ABC----
8 -> 0x7F  ABCDEFG
9 -> 0x6F  ABCD-FG
```

`ABCDEF-`(숫자 0)는 a~f가 켜지고 g가 꺼진 모양, 즉 가운데 막대만 빠진 "0"입니다.

## 자주 하는 실수

### Q. 자리가 겹쳐 보이거나 흐릿한 잔상이 생겨요.
A. 세그먼트 패턴을 바꾸기 전에 자리를 끄지 않아서입니다. 자리가 켜진 상태로 패턴이 바뀌면 그 짧은 순간 엉뚱한 숫자가 그 자리에 비칩니다. 반드시 **끄기(DIG_PORT=0) → 패턴 쓰기 → 자리 켜기** 순서를 지키세요.

### Q. 한 자리만 밝고 나머지는 어두워요.
A. 자리당 유지 시간(`delay`)이 너무 길거나, 한 자리에서만 멈춰 있는 겁니다. 4자리를 충분히 빠르게(보통 자리당 1~5ms, 전체 갱신 50Hz 이상) 돌아야 모두 같은 밝기로 보입니다. 시뮬에서는 P0가 4비트를 모두 도는지 확인하세요.

### Q. 표시가 거꾸로 나와요(4321).
A. `digits[]`의 순서나 자리 선택 비트(`1<<d`)의 대응이 뒤집힌 경우입니다. `digits[0]`을 어느 물리적 자리에 연결할지 약속을 정하고 일관되게 맞추세요.

### Q. 공통 양극 디스플레이를 샀는데 다 켜지거나 다 꺼져요.
A. 이 코드는 공통 음극(비트=1 ON) 가정입니다. 공통 양극이면 ON/OFF가 반대라 세그먼트 코드를 반전(`~`)해야 합니다. 과제 1에서 변환을 직접 만들어 봅니다.

### Q. 룩업 테이블을 RAM이 아니라 ROM에 두고 싶어요.
A. Keil C51에서는 `unsigned char code SEG_CODE[10] = {...};` 처럼 `code` 키워드를 붙이면 프로그램 메모리(ROM)에 놓여 RAM을 아낍니다. 이 트랙은 이식성을 위해 일반 `const`만 썼지만, 8051에서 RAM이 빠듯하면 `code`가 유용합니다.

## 정리
- 7-세그먼트는 a~g(+dp)로 숫자를 그리며, 공통 음극은 비트=1이 ON이다.
- 숫자→패턴은 룩업 테이블 `SEG_CODE[n]`로 빠르게 변환한다.
- 자리를 늘려도 세그먼트선을 공유하면 핀을 아낀다(한 순간 한 자리).
- 멀티플렉싱: 빠르게 자리를 번갈아 켜면 잔상으로 모두 켜 보인다.
- 잔상 방지: 끄기 → 패턴 쓰기 → 자리 켜기 순서를 지킨다.
