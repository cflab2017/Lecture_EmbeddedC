두 문제 모두 **순수 로직(PC `gcc` 검증)** 입니다. 7-세그먼트 표시 자체는 시뮬레이터에 그래픽이 없으므로, 표시의 "두뇌"에 해당하는 코드 변환·자리 분해를 PC에서 정확히 채점합니다.

```
gcc main.c -o prog && ./prog
```

## 문제 1 — 공통 양극 코드 표 만들기
- 파일명: `homework_01/main.c`
- 핵심 개념: 공통 음극 ↔ 공통 양극, 비트 반전, 8비트 마스킹
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 강의의 공통 음극 코드 표(`0x3F, 0x06, …, 0x6F`)로부터, **공통 양극** 코드 표를 만들어 0~9를 출력한다.
- 공통 양극은 ON/OFF가 반대(비트=0이 ON)이므로, 양극 코드 = 음극 코드의 비트 반전이다.
- `~` 결과는 `unsigned char`로 잘라 8비트 두 자리 16진수로 출력한다. 형식 `n -> 0xNN`은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
0 -> 0xC0
1 -> 0xF9
2 -> 0xA4
3 -> 0xB0
4 -> 0x99
5 -> 0x92
6 -> 0x82
7 -> 0xF8
8 -> 0x80
9 -> 0x90
```

### 힌트
- `unsigned char anode = (unsigned char)~SEG_CC[n];`
- 캐스팅을 빼면 `~`가 int로 확장돼 `0xFFFFFFC0`처럼 출력된다. `(unsigned char)` 또는 `& 0xFF`로 막는다.
- 출력은 `printf("%d -> 0x%02X\n", n, anode);`.

## 문제 2 — 표시할 숫자를 자리별로 분해
- 파일명: `homework_02/main.c`
- 핵심 개념: 나눗셈/나머지로 자릿수 추출, 룩업 테이블 인덱싱
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 변수 `n = 2025`를 천/백/십/일 네 자리로 분해한다. `digit[0]`이 천의 자리(가장 왼쪽).
- 각 자리의 숫자와 그 공통 음극 세그먼트 코드(`SEG_CODE[자리값]`)를 함께 출력한다.
- 형식 `digit[i]=값 code=0xNN`은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
digit[0]=2 code=0x5B
digit[1]=0 code=0x3F
digit[2]=2 code=0x5B
digit[3]=5 code=0x6D
```

### 힌트
- 천 = `n/1000%10`, 백 = `n/100%10`, 십 = `n/10%10`, 일 = `n%10`.
- `%10`을 빼먹으면 백의 자리에 20 같은 두 자리 값이 들어간다.
- 추출한 자리값을 그대로 `SEG_CODE[]`의 인덱스로 쓴다(0~9 범위).
