# 06. 8051 구조·SFR·reg52.h

지금까지(1~5편)는 보드 없이 C 문법과 비트·포인터 개념을 다졌습니다. 이번 편부터는 **실제 칩** 위에 올라섭니다. 첫 주인공은 임베디드 교육의 고전 **8051**(여기서는 AT89C52)입니다. 8051은 구조가 단순하고 레지스터가 적어, "C로 하드웨어를 어떻게 만지는가"를 배우기에 가장 좋은 칩입니다.

이번 편의 핵심은 **SFR(Special Function Register, 특수 기능 레지스터)** 입니다. 8051의 포트·타이머·시리얼·인터럽트는 모두 SFR로 제어합니다. Keil C51이 제공하는 `<reg52.h>`가 이 SFR들을 미리 이름으로 선언해 두어, 우리는 `P1 = 0x0F;`처럼 변수에 대입하듯 하드웨어를 다룰 수 있습니다. 이번 편은 포트 SFR에 값을 쓰는 두 가지 방법(포트 전체 / 개별 비트 `sbit`)을 익히고, 7편의 LED 점멸로 이어집니다.

> 시뮬레이터 참고: 이 트랙의 8051 예제는 μVision 내장 시뮬레이터에서 검증합니다. 실제 보드는 필요 없으며, 포트 상태는 Peripherals → I/O-Ports 창에서 확인합니다.

## 학습 목표
- 8051의 기본 구조(CPU, ROM/RAM, 4개 포트, 타이머, 시리얼, 인터럽트)를 개략한다.
- SFR이 무엇이고 메모리 어디(0x80~0xFF)에 위치하는지 설명한다.
- `<reg52.h>`가 SFR을 어떻게 이름으로 노출하는지 이해한다.
- 포트 SFR에 전체 쓰기(`P1 = ...`)와 비트 쓰기(`sbit`)를 구분해 사용한다.
- C51 전용 키워드 `sfr`, `sbit`, `bit`의 쓰임을 안다.

## 대상 환경
- 컴파일러: **Keil C51** (8051 계열 전용).
- 디바이스: **AT89C52** (8051 호환, μVision 시뮬레이터 지원).
- 검증: μVision 시뮬레이터 (Peripherals → I/O-Ports → Port 1). 순수 로직 검산은 PC `gcc`(`pc_test.c`).

## 핵심 개념

### 1) 8051 한눈에 보기
| 자원 | 내용 |
|------|------|
| CPU | 8비트, 누산기(ACC) 중심 |
| 프로그램 메모리 | 코드 ROM/Flash (AT89C52: 8KB) |
| 데이터 메모리 | 내부 RAM 256바이트(하위 128 + 상위 128), SFR 영역 |
| 입출력 | 4개 8비트 포트 P0~P3 (총 32핀) |
| 타이머 | Timer0/1 (AT89C52는 Timer2 추가) |
| 시리얼 | UART 1채널 |
| 인터럽트 | 외부 2, 타이머, 시리얼 등 |

> `int`가 8051에서는 **16비트**입니다(2편 참고). 그래서 8비트로 충분한 값은 `unsigned char`(또는 `uint8_t`)를 써서 RAM과 코드 크기를 아낍니다.

### 2) SFR — 특수 기능 레지스터
SFR은 주변장치를 제어하는 레지스터로, 내부 RAM 주소 **0x80~0xFF** 영역에 배치됩니다. 포트(P0~P3), 타이머 제어(TCON/TMOD), 시리얼(SCON/SBUF), 인터럽트(IE/IP) 등이 모두 SFR입니다.

| SFR | 주소 | 역할 |
|-----|------|------|
| `P0` | 0x80 | 포트 0 |
| `P1` | 0x90 | 포트 1 |
| `P2` | 0xA0 | 포트 2 |
| `P3` | 0xB0 | 포트 3 |
| `TMOD` | 0x89 | 타이머 모드 |
| `TCON` | 0x88 | 타이머 제어 |
| `SCON` | 0x98 | 시리얼 제어 |
| `IE` | 0xA8 | 인터럽트 인에이블 |

주소가 **0x80, 0x88, 0x90 …처럼 8의 배수(0x_0/0x_8)** 인 SFR은 **비트 주소 지정(bit-addressable)** 이 가능합니다. 즉 `sbit`으로 개별 비트에 접근할 수 있습니다. P0~P3, TCON, SCON, IE, IP가 여기 속합니다.

### 3) `<reg52.h>` — SFR을 이름으로
`<reg52.h>`는 위 SFR들을 미리 선언해 둔 Keil C51 내장 헤더입니다. 내부는 대략 이렇게 생겼습니다.

```c
sfr  P1   = 0x90;   /* P1 이라는 이름이 0x90 번지의 SFR 을 가리킴 */
sfr  TMOD = 0x89;
sbit P1_0 = 0x90;   /* (예시) P1 의 0번 비트 */
```

그래서 우리는 `#include <reg52.h>` 한 줄로 `P1`, `TMOD`, `IE` 같은 이름을 바로 쓸 수 있습니다. 외부 라이브러리는 필요 없습니다.

### 4) 포트에 쓰는 두 가지 방법
**(1) 포트 전체 쓰기** — 8비트를 한 번에.

```c
P1 = 0x0F;   /* P1.0~P1.3 = 1, P1.4~P1.7 = 0 */
```

**(2) 개별 비트 쓰기** — `sbit`으로 비트에 이름을 주고 1개만.

```c
sbit LED0 = P1 ^ 0;   /* P1.0 에 LED0 라는 이름 (파일 범위에 선언) */
LED0 = 1;             /* P1.0 만 1, 나머지 비트는 그대로 */
```

`P1 ^ 0`의 `^`는 XOR이 아니라 **"P1의 0번 비트"** 를 뜻하는 C51 전용 표기입니다. `sbit`은 비트 주소 지정 가능 SFR에만 쓸 수 있습니다.

### 5) C51 전용 키워드 — `sfr`, `sbit`, `bit`
| 키워드 | 의미 | 예 |
|--------|------|----|
| `sfr` | 8비트 SFR 선언 | `sfr P1 = 0x90;` |
| `sbit` | SFR의 한 비트 | `sbit LED = P1 ^ 0;` |
| `bit` | 비트 변수(내부 비트 메모리) | `bit flag;` |

이들은 표준 C에는 없는 Keil C51 확장입니다. 그래서 이 `main.c`는 PC의 `gcc`로는 컴파일되지 않습니다(PC 검산은 `pc_test.c`로 따로 합니다).

## μVision 프로젝트 만들기
1. `Project → New µVision Project…` → 폴더와 이름(예: `c51_sfr`) 지정.
2. Device 선택 창에서 **Atmel → AT89C52** 선택(없으면 Pack Installer에서 8051 디바이스 팩 설치).
3. "Copy STARTUP.A51 to project?" 묻는 창은 **예(Yes)** — 기본 시작 코드 포함.
4. Source Group 1에 `main.c` 추가( 우클릭 → Add New/Existing Item) 후 아래 코드 복붙.
5. `Project → Options for Target → Debug` 탭에서 **Use Simulator** 선택.
6. Rebuild(F7) → Start/Stop Debug Session(Ctrl+F5).
7. `Peripherals → I/O-Ports → Port 1` 창을 열어 비트 상태를 관찰.

## 예제로 보기

### 예제 1 — `main.c` : SFR에 접근하는 두 방법
포트 전체 쓰기와 `sbit` 비트 쓰기를 함께 보여 줍니다.

```c
#include <reg52.h>

/*
 * 06강 예제 1 — SFR(특수 기능 레지스터)에 접근하는 두 가지 방법.
 * 대상: AT89C52 (8051 호환), 컴파일러: Keil C51.
 *
 *  (1) 포트 전체에 한 번에 쓰기:  P1 = 0x0F;
 *  (2) 개별 비트에 이름을 붙여 쓰기: sbit 로 선언 후 LED0 = 0;
 *
 * reg52.h 안에서 P1 은 'sfr P1 = 0x90;' 으로 미리 선언돼 있어,
 * 우리는 P1 이라는 이름으로 0x90 번지의 포트 SFR 에 바로 접근한다.
 */

/* P1 포트의 개별 비트에 의미 있는 이름을 붙인다 (P1 은 비트 주소 가능 SFR). */
sbit LED0 = P1 ^ 0;    /* P1.0 */
sbit LED7 = P1 ^ 7;    /* P1.7 */

void main(void)
{
    /* (1) 포트 전체 쓰기: P1.0~P1.3 = 1, 나머지 = 0 */
    P1 = 0x0F;

    /* (2) 개별 비트 쓰기: 나머지 비트는 그대로 두고 한 비트만 */
    LED0 = 0;          /* P1.0 = 0 → 0x0E */
    LED7 = 1;          /* P1.7 = 1 → 0x8E */

    /* 임베디드 main 은 종료하지 않는다. P1 값을 유지하며 대기. */
    while (1) {
        /* 시뮬레이터 Port 1 창에서 P1 = 0x8E 를 확인한다. */
    }
}
```

### 예제 2 — `pc_test.c` : 최종 포트 값 PC 검산
SFR을 일반 변수로 흉내 내어, 위 조작의 결과(0x8E)를 PC에서 검산합니다.

```c
#include <stdio.h>

/*
 * 06강 예제 2 — 강의 예제(main.c)가 만드는 최종 포트 값을 PC 에서 검산한다.
 * 실제 칩에서는 P1 이 SFR(0x90 번지)이지만, 여기서는 일반 변수로 흉내내
 * "P1 = 0x0F 후 P1.0 을 0, P1.7 을 1 로" 했을 때의 결과만 확인한다.
 */
int main(void)
{
    unsigned char P1 = 0x0F;          /* 포트 전체 쓰기 */

    P1 = (unsigned char)(P1 & ~(1u << 0));   /* LED0 = 0 → P1.0 clear */
    P1 = (unsigned char)(P1 | (1u << 7));    /* LED7 = 1 → P1.7 set   */

    printf("P1 = 0x%02X\n", P1);      /* 0x8E */

    return 0;
}
```

## 시뮬레이터로 확인하기
- Ctrl+F5로 디버그 세션 시작 후, F5로 실행(또는 F11로 한 줄씩).
- `Peripherals → I/O-Ports → Port 1` 창에서 P1 비트를 본다.
- **예상 동작**: `P1 = 0x0F`(0000 1111) → `LED0 = 0`(0000 1110) → `LED7 = 1`(1000 1110). 최종 **P1 = 0x8E**. Port 1 창에서 비트7과 비트1~3이 켜진 패턴(1000 1110)이 보인다.

## PC에서 검증하기
순수 로직 검산만 PC에서 가능합니다(디바이스 `main.c`는 C51 전용).

```
gcc pc_test.c -o pc_test && ./pc_test
```

예상 출력(stdout):

```
P1 = 0x8E
```

## 자주 하는 실수

### Q. `gcc`로 `main.c`를 컴파일하니 `reg52.h`가 없다고 나와요.
A. `<reg52.h>`, `sfr`, `sbit`은 Keil C51 전용입니다. 디바이스 `main.c`는 μVision에서만 빌드됩니다. PC에서는 `pc_test.c`(표준 C)로 로직만 검산하세요.

### Q. `sbit LED = P1 ^ 0;`을 함수 안에 썼더니 오류가 나요.
A. `sfr`/`sbit` 선언은 **파일 범위(전역)** 에 둬야 합니다. 함수 안에서는 선언할 수 없습니다. `main` 위쪽에 선언하세요.

### Q. `P1 = 0x0F` 했는데 입력이 안 읽혀요.
A. 이번 편은 출력 쓰기만 다룹니다. 8051에서 포트를 입력으로 읽으려면 먼저 해당 핀에 1을 써서 입력 모드로 만들어야 합니다(준쌍방향 포트). 입력 읽기는 8편에서 다룹니다.

### Q. 값이 큰 카운터에 `int`를 썼더니 이상해요.
A. 8051에서 `int`는 16비트입니다. 8비트면 충분한 포트 값/카운터는 `unsigned char`를 쓰세요. 큰 수가 필요하면 `unsigned int`(16비트)나 `unsigned long`(32비트)을 명시합니다.

## 정리
- 8051은 단순한 8비트 MCU로, 주변장치를 SFR(0x80~0xFF)로 제어한다.
- `<reg52.h>`가 `P1`, `TMOD`, `IE` 등 SFR을 이름으로 제공한다.
- 포트는 전체 쓰기(`P1 = 0x0F`)와 비트 쓰기(`sbit`)로 다룬다.
- `sfr`/`sbit`/`bit`은 C51 전용 키워드라 PC `gcc`로는 안 된다(로직은 `pc_test.c`로 검산).
- 8비트로 충분한 값엔 `unsigned char`를 써서 자원을 아낀다.
