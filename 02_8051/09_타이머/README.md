# 09. 타이머0/1 과 지연

7편의 소프트웨어 지연(`for` 빈 루프)은 간단하지만 **부정확**합니다. 클럭, 컴파일러 최적화, 명령어 수에 따라 시간이 들쭉날쭉하고, 그동안 CPU는 아무것도 못 합니다. 정확한 시간이 필요하면 **하드웨어 타이머**를 씁니다. 타이머는 CPU와 독립적으로 클럭을 세는 카운터라, "정확히 50ms"를 만들 수 있습니다.

이번 편은 8051의 **Timer0/Timer1**을 16비트 모드로 설정해 정확한 지연을 만듭니다. 핵심은 세 가지입니다. **TMOD**로 모드를 고르고, **TH/TL**에 리로드 값을 넣어 원하는 시간 뒤 오버플로가 나게 하고, **TF**(오버플로 플래그)를 폴링해 그 시점을 잡습니다. 리로드 값 계산은 PC `gcc`로 검증합니다.

## 학습 목표
- 소프트웨어 지연 대비 하드웨어 타이머의 장점을 설명한다.
- `TMOD`로 Timer0/1의 모드를 설정한다(Mode 1 = 16비트).
- 원하는 지연에 맞는 리로드 값(TH/TL)을 계산한다.
- `TR`(시작)·`TF`(오버플로 플래그)로 폴링 지연을 구현한다.
- 16비트 한계를 넘는 긴 지연은 오버플로를 여러 번 세서 만든다.

## 대상 환경
- 컴파일러: **Keil C51**.
- 디바이스: **AT89C52** (μVision 시뮬레이터). 12MHz 크리스털(머신사이클 1µs) 가정.
- 검증: μVision 시뮬레이터 (레지스터 창에서 TH0/TL0/TF0 관찰). 리로드 계산은 PC `gcc`(`pc_test.c`).

## 핵심 개념

### 1) 8051 타이머의 동작
타이머는 머신사이클마다 1씩 증가하는 카운터입니다. 표준 8051은 **머신사이클 = 12 클럭**이라, 12MHz 크리스털이면 머신사이클이 1µs입니다. 16비트 카운터(TH:TL)가 `0xFFFF`에서 한 번 더 증가하면 `0x0000`으로 넘어가며 **오버플로 플래그(TF)** 가 1이 됩니다.

> 미리 큰 값을 넣어 두면(리로드) 더 빨리 오버플로가 납니다. "65536 - 원하는 카운트"를 넣으면, 그 카운트만큼 센 뒤 오버플로가 발생합니다.

### 2) TMOD — 타이머 모드 레지스터
TMOD의 하위 4비트는 Timer0, 상위 4비트는 Timer1을 설정합니다.

| 비트 | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|------|---|---|---|---|---|---|---|---|
| 의미 | GATE1 | C/T1 | M1 | M0 | GATE0 | C/T0 | M1 | M0 |
| 대상 | ←─ Timer1 ─→ | | | | ←─ Timer0 ─→ | | | |

| M1 M0 | 모드 | 설명 |
|-------|------|------|
| 0 0 | Mode 0 | 13비트 |
| 0 1 | **Mode 1** | **16비트(이 편에서 사용)** |
| 1 0 | Mode 2 | 8비트 자동 리로드 |
| 1 1 | Mode 3 | 분할 |

- Timer0 Mode1: `TMOD = 0x01` (하위 니블 `0001`).
- Timer1 Mode1: `TMOD = 0x10` (상위 니블 `0001`).

다른 타이머 설정을 보존하려면 통째로 대입하지 말고 마스크로: `TMOD &= 0xF0; TMOD |= 0x01;`(Timer0만 건드림).

### 3) 리로드 값 계산
12MHz(1µs/카운트)에서 50ms 지연을 만들려면:

```
필요한 카운트 = 50ms ÷ 1µs = 50000
리로드 값     = 65536 - 50000 = 15536 = 0x3CB0
TH0 = 0x3C (상위), TL0 = 0xB0 (하위)
```

일반식: `reload = 65536 - (지연µs)`, `TH = reload >> 8`, `TL = reload & 0xFF`.

### 4) TCON — 시작과 오버플로 플래그
| 비트 | 이름 | 역할 |
|------|------|------|
| TR0 / TR1 | Timer Run | 1이면 타이머 동작 |
| TF0 / TF1 | Timer Flag | 오버플로 시 1 (소프트웨어가 클리어) |

`reg52.h`가 이들을 `sbit`으로 제공하므로 `TR0 = 1;`, `while (TF0 == 0);`처럼 바로 씁니다.

### 5) 폴링 지연의 흐름
```
TMOD 설정 → TH/TL 리로드 → TF=0 → TR=1(시작)
 → while(TF==0) 대기 → TR=0(정지) → TF=0(클리어)
```

> 폴링은 대기 중 CPU가 묶입니다. 대기 동안 다른 일을 하려면 **타이머 인터럽트**(10편)를 씁니다.

## 예제로 보기

### 예제 1 — `main.c` : 타이머 폴링 지연으로 점멸
Timer0 Mode1으로 50ms를 만들어 LED를 토글합니다.

```c
#include <reg52.h>

/*
 * 09강 예제 1 — 하드웨어 타이머로 정확한 지연을 만들어 LED 점멸.
 * 대상: AT89C52, 컴파일러: Keil C51. 12MHz 크리스털 가정(머신사이클 1us).
 *
 * Timer0 를 Mode 1(16비트)로 쓴다. 카운터가 0xFFFF 를 넘으면 TF0(오버플로
 * 플래그)가 1 이 된다. 50ms 뒤 오버플로가 나도록 리로드 값을 미리 넣는다:
 *   필요한 카운트 = 50ms / 1us = 50000
 *   리로드 = 65536 - 50000 = 15536 = 0x3CB0  → TH0=0x3C, TL0=0xB0
 */

sbit LED = P1 ^ 0;     /* 액티브 로우 LED */

/* Timer0 Mode1 로 약 50ms 지연(폴링 방식). */
static void delay_50ms(void)
{
    TMOD &= 0xF0;          /* Timer0 설정 비트만 0 으로(Timer1 설정은 보존) */
    TMOD |= 0x01;          /* Timer0 = Mode 1 (16비트) */

    TH0 = 0x3C;            /* 리로드 상위 바이트 */
    TL0 = 0xB0;            /* 리로드 하위 바이트 */

    TF0 = 0;               /* 오버플로 플래그 클리어 */
    TR0 = 1;               /* Timer0 시작 */
    while (TF0 == 0) {     /* 오버플로(=50ms 경과)까지 대기 */
    }
    TR0 = 0;               /* Timer0 정지 */
    TF0 = 0;               /* 다음을 위해 플래그 클리어 */
}

void main(void)
{
    while (1) {
        LED = (bit)!LED;   /* LED 토글 */
        delay_50ms();      /* 50ms 간격 → 약 10Hz 점멸 */
    }
}
```

### 예제 2 — `pc_test.c` : 리로드 값 계산 검증
여러 지연에 대해 리로드와 TH/TL을 계산해 봅니다.

```c
#include <stdio.h>

/*
 * 09강 예제 2 — 타이머 리로드 값 계산을 PC 에서 검증한다.
 * 12MHz(머신사이클 1us) 기준, 원하는 지연(us)에 필요한 16비트 리로드와
 * TH/TL 바이트를 구한다. Mode1 은 16비트라 1us 단위로 최대 65536us.
 *   reload = 65536 - us,  TH = reload>>8,  TL = reload & 0xFF
 */
int main(void)
{
    unsigned int delays_us[] = { 1000, 5000, 50000 };
    int n = (int)(sizeof(delays_us) / sizeof(delays_us[0]));
    int k;

    for (k = 0; k < n; k++) {
        unsigned int us = delays_us[k];
        unsigned int reload = (unsigned int)(65536UL - us);
        unsigned char th = (unsigned char)(reload >> 8);
        unsigned char tl = (unsigned char)(reload & 0xFF);
        printf("%5u us -> reload=%5u  TH0=0x%02X TL0=0x%02X\n",
               us, reload, th, tl);
    }

    return 0;
}
```

## 시뮬레이터로 확인하기
- `Peripherals → Timer → Timer 0` 창에서 모드·실행 상태를, 레지스터 창에서 TH0/TL0/TF0를 본다.
- F5로 실행하면 P1.0이 주기적으로 토글된다.
- **예상 동작**: Xtal 12MHz 설정 시, 디버거의 사이클/시간 표시(`Registers → sec`)로 약 50ms마다 토글됨을 확인. Logic Analyzer에 P1.0을 추가하면 약 10Hz 구형파가 보인다.

## 자주 하는 실수

### Q. `TMOD = 0x01`로 통째로 대입했더니 Timer1 설정이 날아갔어요.
A. TMOD는 두 타이머를 함께 담습니다. 통째로 대입하면 다른 타이머 설정이 지워집니다. 해당 타이머 비트만 바꾸세요: `TMOD &= 0xF0; TMOD |= 0x01;`(Timer0). Timer1은 `TMOD &= 0x0F; TMOD |= 0x10;`.

### Q. 지연 시간이 예상과 달라요.
A. ① 크리스털 주파수가 12MHz가 아니면 머신사이클이 1µs가 아닙니다(리로드 재계산). ② 시뮬레이터의 Xtal 설정을 실제와 맞췄는지 확인. ③ 일부 8051 변종은 "1T" 모드(머신사이클=1클럭)라 6~12배 빠릅니다.

### Q. 첫 지연은 되는데 그다음부터 즉시 통과해요.
A. `TF`(오버플로 플래그)를 다시 클리어하지 않아서입니다. Mode1은 자동 리로드가 아니므로, 매 지연마다 TH/TL을 다시 넣고 TF를 0으로 클리어해야 합니다.

### Q. 1초처럼 긴 지연이 안 만들어져요.
A. 16비트(1µs 단위)는 한 번에 최대 ~65ms입니다. 더 길게는 짧은 오버플로(예: 50ms)를 **여러 번 세서** 만듭니다(1초 = 50ms × 20). 과제 2에서 이 계산을 연습합니다.

## 정리
- 하드웨어 타이머는 CPU와 독립적으로 정확한 시간을 만든다.
- TMOD로 모드 설정(Mode1=16비트), Timer0은 하위 니블·Timer1은 상위 니블.
- 리로드 = 65536 − 카운트, TH=상위·TL=하위.
- TR로 시작, TF 폴링으로 오버플로 시점을 잡고, 매번 TF를 클리어한다.
- 16비트 한계를 넘는 지연은 오버플로 횟수를 세서 만든다.
