# 과제 - 09. 타이머0/1 과 지연

문제 1은 **C51(AT89C52, μVision 시뮬레이터)**, 문제 2는 **순수 로직(PC `gcc` 검증)** 입니다.

---

## 문제 1 — Timer1으로 LED 점멸
- 파일명: `homework_01/main.c`
- 핵심 개념: Timer1 설정(TMOD 상위 니블), TR1/TF1 폴링
- 플랫폼: C51 (AT89C52)

### 요구사항
- **Timer1**을 Mode 1(16비트)으로 설정해 약 50ms 지연을 만들고, P1.0 LED를 토글한다.
- 12MHz 가정(머신사이클 1µs), 리로드 = 65536 − 50000 = 0x3CB0 → TH1=0x3C, TL1=0xB0.

### 예상 동작 (시뮬레이터)
- P1.0 LED가 약 50ms 간격(≈10Hz)으로 토글된다(예제 1과 같은 주기, 단 Timer1 사용).

### 힌트
- Timer1은 TMOD 상위 니블: `TMOD &= 0x0F; TMOD |= 0x10;`.
- 제어/플래그도 Timer1용: `TR1`, `TF1`. 매 지연마다 TH1/TL1 재설정 + TF1 클리어.

## 문제 2 — 긴 지연에 필요한 오버플로 횟수 (PC)
- 파일명: `homework_02/main.c`
- 핵심 개념: 16비트 한계, 오버플로 누적
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 오버플로 1회를 50ms로 가정할 때, 목표 시간 `100, 500, 1000`(ms)에 필요한 **오버플로 횟수**를 계산해 출력한다.
- 출력 형식은 "예상 출력"과 글자 단위로 일치해야 한다(`%4lu ms`, `%2lu overflows`).

### 예상 출력
```
 100 ms ->  2 overflows (50ms each)
 500 ms -> 10 overflows (50ms each)
1000 ms -> 20 overflows (50ms each)
```

### 힌트
- 단위를 µs로 환산: `total_us = ms * 1000`, `overflows = total_us / 50000`.
- 정수 나눗셈이라 딱 떨어지는 값들로 골랐다.
