## 문제 1 — 함수 포인터로 드라이버 교체
- 파일명: `homework_01/main.c`
- 핵심 개념: 함수 포인터 기반 HAL, 런타임 구현 교체
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- `void (*set)(int on)` 함수 포인터를 가진 드라이버 구조체를 정의한다.
- 두 구현을 만든다: A는 `A: ON/OFF`, B는 `B: [*]`(켜짐)/`B: [.]`(꺼짐)로 출력.
- 같은 `app_run`이 패턴 `{1,0,1,1}`을 두 드라이버로 각각 구동한다.
- 형식은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
-- driver A --
A: ON
A: OFF
A: ON
A: ON
-- driver B --
B: [*]
B: [.]
B: [*]
B: [*]
```

### 힌트
- `typedef struct { void (*set)(int); } led_driver_t;`
- `app_run(const led_driver_t *drv)`에서 `drv->set(pat[i])` 호출.
- 같은 app_run에 드라이버만 바꿔 넘기면 동작이 달라진다(이식성의 런타임 버전).

## 문제 2 — 이식 가능한 map() 유틸리티
- 파일명: `homework_02/main.c`
- 핵심 개념: 하드웨어 비의존 유틸 함수, 범위 변환
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 값 `x`를 입력 범위 `[in_lo, in_hi]`에서 출력 범위 `[out_lo, out_hi]`로 선형 변환하는 `map()`을 구현한다.
- `map(x) = (x - in_lo) * (out_hi - out_lo) / (in_hi - in_lo) + out_lo`.
- ADC 범위 `0~4095`를 퍼센트 `0~100`으로 변환해, `x = {0,1024,2048,3072,4095}`에 대해 출력한다.
- 형식 `map N -> P`은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
map 0 -> 0
map 1024 -> 25
map 2048 -> 50
map 3072 -> 75
map 4095 -> 100
```

### 힌트
- 곱셈을 먼저, 나눗셈을 나중에(정수 오차 최소화). 오버플로가 걱정되면 `long` 사용.
- 이런 순수 유틸은 칩과 무관하므로 HAL 위/아래 어디서든 재사용된다.
