## 문제 1 — 덮어쓰기(overwrite) 링버퍼
- 파일명: `homework_01/main.c`
- 핵심 개념: 가득 찼을 때 가장 오래된 데이터 폐기, wrap
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 크기 4(용량 3)의 링버퍼를 만든다. 가득 찬 상태에서 push하면 **가장 오래된 값을 버리고**(그 값을 표시) 새 값을 넣는다(로그 버퍼 정책).
- `A`~`E`를 차례로 push한 뒤 전체를 drain한다.
- 형식은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
push A -> ok
push B -> ok
push C -> ok
push D -> overwrite (dropped A)
push E -> overwrite (dropped B)
drain: CDE
```

### 힌트
- 가득 판정: `((head+1)&MASK)==tail`. 가득이면 `tail`을 한 칸 전진시켜 가장 오래된 값을 버린다.
- 크기 4는 2의 거듭제곱이라 `& 3`으로 wrap.

## 문제 2 — 수신 스트림에서 줄 단위 조립
- 파일명: `homework_02/main.c`
- 핵심 개념: 라인 버퍼, 구분자(`\n`)까지 누적 (UART 명령 수신 패턴)
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 문자 스트림 `"go\nstop\nhi\n"`을 한 글자씩 읽어, `\n`을 만날 때마다 그때까지 모은 줄을 출력한다.
- 형식 `LINE: <줄>`은 예상 출력과 글자 단위로 일치해야 한다(개행 문자는 줄에 포함하지 않음).

### 예상 출력
```
LINE: go
LINE: stop
LINE: hi
```

### 힌트
- 라인 버퍼에 글자를 누적하다가 `\n`이면 널 종료 후 출력, 인덱스를 0으로 리셋.
- 이 패턴이 24편에서 UART로 받은 명령을 한 줄씩 해석하는 토대가 된다.
