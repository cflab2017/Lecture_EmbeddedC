# 과제 - 24. 종합 — UART 명령으로 LED·타이머 제어

문제 1은 **순수 로직(PC `gcc`)**, 문제 2는 **MDK(STM32F103C8, μVision 시뮬레이터)** 입니다.

---

## 문제 1 — 인자 있는 명령 파서
- 파일명: `homework_01/main.c`
- 핵심 개념: 명령+숫자 인자 파싱, 범위 클램프
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- `on`, `off`는 인자 없이 인식한다(값 0).
- `dim N`은 0~100 밝기 인자를 받는다. 100을 넘으면 100으로 **클램프**한다.
- 그 외는 `UNKNOWN`.
- 형식 `"줄" -> 명령 (값)`은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
"on" -> ON (0)
"off" -> OFF (0)
"dim 50" -> DIM (50)
"dim 100" -> DIM (100)
"dim 200" -> DIM (100)
"foo" -> UNKNOWN (0)
```

### 힌트
- `dim ` 접두사를 확인하고 그 뒤 숫자를 직접 파싱(`v = v*10 + (ch-'0')`).
- 파싱한 값이 100보다 크면 100으로 자른다.

## 문제 2 — UART로 숫자를 10진수로 출력
- 파일명: `homework_02/main.c`
- 핵심 개념: 정수 → 10진 문자열, UART 송신
- 플랫폼: MDK (STM32F103C8)

### 요구사항
- USART1(9600 8N1)을 초기화하고, 16비트 정수를 10진수 텍스트로 보내는 `send_dec()`를 구현한다.
- `0`, `42`, `1000`을 공백으로 구분해 보내고 줄을 마친다(`\r\n`).
- 상태값(예: 카운터, ADC 값)을 사람이 읽게 보낼 때 쓰는 핵심 헬퍼다.

### 예상 동작 (시뮬레이터)
- UART #1 창에 `0 42 1000`이 출력되고 줄바꿈된다.

### 힌트
- 0은 특수 처리(`'0'` 한 글자). 그 외는 `v % 10`으로 끝자리부터 뽑아 버퍼에 담고 **역순**으로 보낸다.
- `while (v > 0) { buf[i++] = '0' + v % 10; v /= 10; }` 후 거꾸로 송신.
