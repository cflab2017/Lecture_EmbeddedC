# 과제 - 20. ADC 아날로그 입력

문제 1은 **순수 로직(PC `gcc`)**, 문제 2는 **MDK(STM32F103C8, μVision 시뮬레이터)** 입니다.

---

## 문제 1 — ADC 카운트를 퍼센트(%)로
- 파일명: `homework_01/main.c`
- 핵심 개념: 스케일 변환, 정수 나눗셈 순서
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- ADC 카운트(0~4095)를 0~100% 로 환산해 출력한다. `pct = adc × 100 / 4095`.
- 입력은 `{0, 1024, 2048, 3071, 4095}`.
- 형식 `ADC=N -> P%`은 예상 출력과 글자 단위로 일치해야 한다.

### 예상 출력
```
ADC=0 -> 0%
ADC=1024 -> 25%
ADC=2048 -> 50%
ADC=3071 -> 74%
ADC=4095 -> 100%
```

### 힌트
- `pct = (uint32_t)adc * 100u / 4095u;` (곱한 뒤 나누기).
- 3071은 정수 절삭으로 75가 아니라 **74**가 된다(절삭의 자연스러운 결과).

## 문제 2 — ADC 임계값으로 LED 제어
- 파일명: `homework_02/main.c`
- 핵심 개념: ADC 채널1(PA1), 임계 비교, LED 제어
- 플랫폼: MDK (STM32F103C8)

### 요구사항
- ADC1 채널1(PA1)을 읽어, 값이 2048(약 절반)보다 크면 PC13 LED를 켜고 아니면 끈다(액티브 로우).
- 보정 등 변환 시퀀스는 예제와 동일하게 한다.

### 예상 동작 (시뮬레이터)
- `SQR3`의 첫 채널이 1, PA1이 아날로그 입력(0x0).
- ADC 값 > 2048이면 PC13=0(LED ON), 아니면 PC13=1(OFF). Watch로 `adc_val`과 GPIOC ODR 확인.

### 힌트
- 채널 변경: `ADC1->SQR3 = 1u;` 그리고 PA1을 아날로그 입력으로(`GPIOA->CRL` 비트[7:4] 클리어).
- LED: `if (v > 2048u) GPIOC->BRR = (1u<<13); else GPIOC->BSRR = (1u<<13);`.
