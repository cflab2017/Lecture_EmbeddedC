# 과제 - 19. USART 시리얼 통신

문제 1은 **순수 로직(PC `gcc`)**, 문제 2는 **MDK(STM32F103C8, μVision 시뮬레이터)** 입니다.

---

## 문제 1 — 바이트를 16진수 ASCII 두 글자로
- 파일명: `homework_01/main.c`
- 핵심 개념: 니블 → 16진 문자, UART 디버그 출력 헬퍼
- 플랫폼: 공통 (PC `gcc`)

### 요구사항
- 바이트 값을 대문자 16진수 두 글자(예: `0x2A` → `"2A"`)로 변환해 출력한다.
- 입력 바이트는 `{0x00, 0x2A, 0xFF, 0x7E, 0xA5}`.
- 형식 `0xNN -> "XX"`은 예상 출력과 글자 단위로 일치해야 한다(둘 다 두 자리 대문자).

### 예상 출력
```
0x00 -> "00"
0x2A -> "2A"
0xFF -> "FF"
0x7E -> "7E"
0xA5 -> "A5"
```

### 힌트
- 상위 니블 `(b >> 4) & 0xF`, 하위 니블 `b & 0xF`.
- 니블 → 문자: `n < 10 ? '0'+n : 'A'+(n-10)`.
- 출력 형식의 양쪽 16진수를 모두 `%02X`로 맞추면 편하다.

## 문제 2 — 소문자를 대문자로 바꿔 에코
- 파일명: `homework_02/main.c`
- 핵심 개념: USART 수신/송신, 문자 변환
- 플랫폼: MDK (STM32F103C8)

### 요구사항
- USART1을 9600 8N1로 초기화하고, 시작 시 `READY\r\n`을 보낸다.
- 받은 문자가 소문자(`a`~`z`)면 대문자로 바꿔 에코하고, 그 외는 그대로 에코한다.

### 예상 동작 (시뮬레이터)
- UART #1 창에 `READY`가 먼저 출력된다.
- `abc123`을 입력하면 `ABC123`이 에코된다(소문자만 대문자로).

### 힌트
- 소문자 판정: `if (c >= 'a' && c <= 'z') c = c - 'a' + 'A';`.
- 예제의 `usart1_init` / `usart1_send_char`를 그대로 활용.
